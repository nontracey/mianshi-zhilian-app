# 面试智练竞品学习与视觉改进方案

> 日期：2026-05-29  
> 目标：参考同类型产品的交互模式，评估面试智练如何变得更好看、更好用、更适合跨平台使用，并提供界面参考图。

## 1. 参考产品与可借鉴点

### Google Interview Warmup

参考地址：[Interview Warmup - Grow with Google](https://grow.google/certificates/interview-warmup/)

可借鉴点：

1. 先让用户“说出来”，再把语音转写成文本。
2. 不直接给用户强评分，而是给答案洞察，例如关键词、话题覆盖、表达模式。
3. 强调隐私：不保存音频和转写内容。
4. 适合做面试练习的低压力入口。

对面试智练的启发：

1. 语音复述默认应先转文字，再让用户编辑确认。
2. AI 反馈不要只有分数，还要有“覆盖了什么、漏了什么、表达有什么问题”。
3. 语音和文本输入应是同一个练习流程，而不是两个割裂功能。

### Yoodli AI Interview Coach

参考地址：[Yoodli AI Interview Coach](https://e.yoodli.ai/use-cases/interview-preparation)

可借鉴点：

1. 面向“说话表现”的实时反馈：语速、停顿、填充词、表达建议。
2. 根据用户回答继续追问。
3. 练习后生成报告，帮助用户持续改进。

对面试智练的启发：

1. 模拟面试需要有追问流，而不是固定题目列表。
2. 语音复述除了技术要点，还可以评估表达结构。
3. 练习结束页应有复盘报告：薄弱知识、表达问题、下一轮建议题。

### interviewing.io

参考地址：[interviewing.io](https://interviewing.io/)

可借鉴点：

1. 真实面试流程感强：coding、system design、behavioral 等场景明确。
2. 反馈强调 actionable feedback，不只是泛泛评价。
3. AI Interviewer 和真人 mock interview 形成梯度。

对面试智练的启发：

1. 练习页应按场景分组：复述、代码题、系统设计、模拟面试、追问训练。
2. AI 反馈要给“下一步怎么练”，例如推荐前置知识点和薄弱 topic。
3. 桌面端适合做“题目 + 答案 + 反馈”的分栏工作台。

### Quizlet Learn

参考地址：[Quizlet Learn](https://quizlet.com/gb/features/learn)

可借鉴点：

1. 自适应学习：根据过去表现聚焦薄弱内容。
2. 多题型切换：选择题、判断题、书写题等循序渐进。
3. 短练习、高反馈、跨设备同步。

对面试智练的启发：

1. 首页要突出“今日复习队列”和“下一步最佳行动”。
2. 推荐策略应可解释：为什么推荐这道题。
3. 复述题可从易到难：提示式复述 -> 完整复述 -> 追问 -> 代码/图解题。

## 2. 面试智练当前视觉问题

结合现有 Flutter App 和内容结构，当前更像 MVP 工具，主要问题是：

1. 首页缺少“学习工作台”的第一眼信息组织：今日要做什么、当前路线在哪、薄弱点是什么不够突出。
2. 目录页操作偏普通列表，面对 298 个 topic 时扫描效率不够。
3. 复述页没有把“模型能力、语音、图片、代码”组织成一个清晰的输入系统。
4. 掌握度页静态展示多，下一步行动少。
5. 设置页有些控件看起来能用，但实际没有完整生效，会降低信任感。
6. 桌面和移动端应有不同布局策略，不能只是同一套页面缩放。
7. 今日复习页还停留在入口卡片，缺少复习队列、间隔复习原因、复习后行动和错题回流。
8. 模拟面试页已经能跑流程，但还不像真实面试工作台：缺少场景选择、追问节奏、计时、面试官反馈和结束复盘。
9. 个人中心、登录页、AI 配置页、更新页等原有页面需要统一到同一套视觉语言，不能只美化核心学习页。

## 3. 推荐视觉方向

产品气质建议定位为：

```text
面向技术面试的 AI 主动回忆工作台
```

关键词：

1. 专业、清晰、高密度。
2. 像 Notion/Linear/Quizlet 的学习工作流结合，而不是营销落地页。
3. 桌面端偏 dashboard + split view。
4. 手机端偏 card + bottom action bar。
5. 状态和能力标签要一眼可见：发布/测试/草稿、文本/图片/语音、熟练/薄弱/待复习。

不建议：

1. 大面积 hero 或装饰插画。
2. 过度渐变和紫蓝科技感。
3. 卡片套卡片。
4. 长按钮挤在移动端列表右侧。

## 4. 全量界面改造目标

所有原有界面都要收敛到同一套“技术面试学习工作台”视觉计划中，避免出现一半新工作台、一半旧 MVP 表单的割裂感。

### 4.1 学习首页 / 工作台

目标：

1. 首屏回答“今天最该做什么”：继续学习、今日复习、薄弱点修复、下一场模拟面试。
2. 桌面端使用三栏工作台：左侧导航，中间主行动区，右侧复盘和建议。
3. 移动端使用纵向任务流：今日任务、继续学习、复习队列、最近反馈。
4. 每个行动卡必须带来源解释，例如“因为 3 天未复习”“上次回答漏掉 GC Roots”。

### 4.2 知识目录

目标：

1. 从普通列表升级为可筛选的知识地图。
2. 支持领域、分类、学习路径、状态、高频、难度、掌握度、是否含代码题、是否含 LeetCode。
3. 每个 topic 行展示状态、面试频率、预计时间、最近得分和下一次复习时间。
4. 桌面端保留高密度表格/列表切换；移动端使用筛选 bottom sheet。

### 4.3 知识详情

目标：

1. 桌面端使用“目录 / 知识卡片 / 复述评估”分栏。
2. 移动端拆成知识阅读、复述、反馈三个连续步骤。
3. 面试官关注点、前置知识、标准回答、追问题必须比普通正文更醒目。
4. 代码、图解、对比表、checklist 使用统一卡片样式和状态标签。

### 4.4 复述练习

目标：

1. 文本、语音、图片、代码是同一条回答输入流程，不做割裂入口。
2. 语音先转写成可编辑文本；图片和代码作为回答上下文进入评分。
3. 反馈结构固定为：总分、概念覆盖、错误点、遗漏点、表达质量、优化回答、下一步练习。
4. 失败时保留草稿、附件和模型选择。
5. 模型选择只展示用户已配置并启用的 AI 模型；App 不提供内置模型。
6. 没有 AI 配置时降级为本地练习：看题、作答、保存草稿、查看参考回答和自评，并提示“配置 AI 后可获得评分”。

### 4.5 今日复习

目标：

1. 独立设计“复习工作台”，不只是从练习页进入一组题。
2. 顶部展示今日复习负载：到期、逾期、薄弱回流、新错题。
3. 队列按原因分组：遗忘曲线到期、低分重练、面试高频、路径前置知识。
4. 每道题展示复习原因、上次得分、上次遗漏点、预计耗时和建议输入模式。
5. 复习完成后给出小结：通过、需要重练、推迟复习、加入模拟面试题池。

### 4.6 模拟面试

目标：

1. 独立设计“面试房间”界面，包含场景、题目、计时、回答区、追问区、面试官反馈。
2. 支持场景选择：基础知识、项目深挖、系统设计、代码题、行为面试、混合面试。
3. 题目流程从固定列表升级为“主问题 -> 追问 -> 澄清 -> 总结”的真实节奏。
4. 结束页提供面试报告：知识薄弱点、表达问题、回答结构、下一轮训练包。
5. 桌面端适合左右分栏；移动端采用全屏单题沉浸式流程。

### 4.7 掌握度

目标：

1. 从静态统计页升级为诊断页。
2. 展示领域掌握度、薄弱分类、长期未复习、高频未掌握、最近退步 topic。
3. 每个图表都要能落到行动：开始复习、补前置、进入模拟面试、查看错因。
4. 第一版优先用条形图、列表和趋势线，复杂雷达图后置。

### 4.8 个人中心 / 设置 / AI 配置 / 登录

目标：

1. 所有设置分组使用同一套 section header、表单控件、能力标签和说明文本。
2. AI 配置突出模型能力、用途标签、默认模型、连接测试结果和隐私/费用提示。
3. 登录页保持轻量，但视觉上与主应用一致：Logo、产品名、同步价值、本地进度合并提示。
4. 主题、字体、密度、语言、同步、更新检查等 no-op 项必须落地或隐藏，避免信任损耗。
5. 未登录用户也能使用个人中心：展示本地游客模式、本地统计、导出入口和同步增强提示。
6. 个人中心首屏设置“账户管理”模块，承载账号信息、头像、昵称、账号数据、绑定邮箱、绑定微信、绑定其他账号和统计。
7. 设置页承载功能配置，例如同步方式、AI 模型、外观、学习策略；账号绑定入口不散落在设置页。
8. 微信绑定、邮箱绑定、其他账号绑定等能力未配置平台 Key 时，保留入口并提示“待开通”。
9. 所有第三方密钥通过环境变量、Worker secrets 或本机未入库配置提供，代码中不写真实 Key。

## 6. 推荐落地优先级

### P0：统一视觉骨架

1. 桌面端：左侧导航 + 顶部状态条 + 主工作区。
2. 手机端：顶部标题 + 内容卡片 + 底部操作条。
3. 统一状态标签：已发布、测试中、草稿。
4. 统一模型能力标签：文本、图片、语音、代码。
5. 统一 topic meta：高频、难度、预计时间、掌握度。

### P1：重做复述页

1. 顶部模型选择器，只展示用户自定义且启用的模型；没有配置时显示配置提示。
2. 输入模式：文本、语音、图片、代码。
3. 语音转写可编辑。
4. 图片附件按模型能力决定是否发送给 AI。
5. 提交后分段展示：分数、遗漏点、错误点、优化回答、下一步。
6. 未配置 AI 时进入本地练习模式，不阻断复习和模拟面试。

### P2：重做首页为学习工作台

1. 今日复习队列。
2. 当前学习路线。
3. 薄弱知识 Top 5。
4. 领域掌握度。
5. 下一步最佳行动。
6. 最近 AI 反馈。

### P3：重做今日复习页

1. 到期、逾期、薄弱回流、新错题分组。
2. 每个复习项展示复习原因、上次得分、遗漏点和预计耗时。
3. 支持一键开始、跳过到明天、降级提示式复述、加入模拟面试题池。
4. 完成后生成今日复习小结，并更新掌握度和下次复习时间。

### P4：重做模拟面试页

1. 面试场景选择：基础知识、项目深挖、系统设计、代码题、行为面试、混合面试。
2. 面试房间：题目、计时、回答输入、追问、草稿、面试官关注点。
3. 结束报告：总分、维度评分、关键问题回放、下一轮训练包。
4. 支持从薄弱点、今日复习、学习路径自动生成一场面试。

### P5：桌面详情页分栏

1. 左侧目录和前置知识。
2. 中间知识卡片。
3. 右侧复述和评估。
4. 大屏启用，小屏自动降级。

### P6：目录页筛选与搜索

1. 全局搜索。
2. 状态筛选按权限展示。
3. 高频、难度、掌握度、含代码题、含 LeetCode 筛选。
4. 移动端筛选用 bottom sheet。

### P7：统一原有页面视觉与可信度

1. 登录、个人中心、AI 配置、掌握度、更新、主题设置统一控件和间距。
2. 所有旧页面补齐空状态、加载状态、错误状态和离线状态。
3. 所有按钮、标签、表单、列表项、分割线、弹窗和 bottom sheet 收敛到统一组件。
4. 没有真实生效的设置项必须实现、标注“即将支持”或移除。

## 7. 面试准备功能池

初版能纳入的功能尽量围绕“准备 -> 练习 -> 反馈 -> 复习 -> 再面试”闭环，不做泛泛题库。

建议进入初版：

1. 今日复习队列：遗忘曲线、低分、逾期、高频未掌握。
2. 主动复述：文本、语音转写、图片附件、代码回答。
3. 追问训练：基于 topic 的 followUps 或 AI 动态追问。
4. 模拟面试：按场景组卷，支持计时、连续答题和结束报告。
5. 面试回答模板：简短版、标准版、深入版，配合面试官关注点。
6. 薄弱点训练包：从错题、低分维度、遗漏概念自动生成。
7. 高频题冲刺：按 `interviewFrequency` 和 `recommendWeight` 排序。
8. 项目深挖训练：STAR/背景-动作-结果、技术决策、取舍、指标、事故复盘，模板和记录都属于用户侧本地数据。
9. 系统设计练习：需求澄清、容量估算、架构草图、瓶颈、权衡。
10. 代码题复述：思路、复杂度、边界条件、测试用例、LeetCode 链接。
11. 面试报告：知识掌握、表达结构、追问稳定性、下一步建议。
12. 准备日程：目标岗位、面试日期、每日新学/复习/模拟面试节奏。

还可以补强的面试准备能力：

1. 准备目标设置：目标岗位、技术栈、面试日期、每日可投入时间、当前基础水平。
2. 面试就绪度：综合掌握度、复习逾期、模拟面试表现、表达稳定性，给出 0-100 的 readiness score。
3. 倒计时计划：根据面试日期自动生成冲刺节奏，例如基础补齐、 高频冲刺、模拟面试、错题回炉。
4. JD/岗位描述解析：用户粘贴岗位描述后，本地生成重点 topic、缺口和练习优先级。
5. 项目深挖库：用户本地维护项目背景、技术决策、指标、难点、故障复盘和个人贡献。
6. 回答版本库：保存用户对高频题的多版答案，支持“初稿 -> AI 修改 -> 面试版”。
7. 错因标签：把低分原因归类为概念缺失、混淆、表达不清、缺少场景、深度不足、代码边界遗漏。
8. 提示阶梯：不会答时先给方向提示、关键词提示、结构提示，最后才显示参考回答。
9. 面试模式切换：练习模式即时反馈；正式模拟模式只在结束后反馈，减少真实面试割裂感。
10. 复盘回放：模拟面试结束后按题回看问题、回答、追问、评分和下一步动作。
11. 轻量仪表盘：连续练习天数、今日完成度、逾期数量、最近 7 天分数趋势。

## 8. 知识契约是否需要改变

结论：本轮不需要改知识契约。

依据 content 项目现有文档和 schema，初版改进需要的公共知识字段已经存在：

1. 复述题：使用现有 `recallPrompts`，其中已有 `id`、`prompt`、`mode`、`expectedMinutes`、`difficulty`。
2. 面试追问：使用现有 `learningCards.type=interviewAnswer` 里的 `followUpQuestions`。
3. 面试回答模板：使用现有 `interviewAnswer` 卡片。
4. 高频和推荐：使用现有 `interviewFrequency`、`recommendWeight`。
5. 前置补学：使用现有 `prerequisites`。
6. 面试官关注点：使用现有 `interviewerFocus`。
7. 学习耗时：使用现有 `estimatedMinutes`。
8. 评分和错因归类：使用现有 `rubric.mustHave`、`rubric.goodToHave`、`rubric.commonMistakes`、`rubric.scoreWeights`。

只有出现字段更名、删除字段、增加必填字段、修改字段类型/语义/枚举、增加新的 `learningCards.type` 或 `recallPrompts.mode` 时，才算知识契约变更。本轮计划不需要这些改动。

## 9. 最终建议

最值得马上做的是“手机端多模态复述”、“桌面学习工作台”、“今日复习工作台”和“模拟面试房间”。

原因：

1. 复述页是产品核心，能直接体现 AI 和多模态能力。
2. 首页工作台能把 298 个 topic 组织成“今天该学什么”，降低用户进入成本。
3. 今日复习把长期留存和掌握度闭环撑起来。
4. 模拟面试把单点练习升级为真实面试准备，能明显提升产品完成度。
5. 同步美化所有原有页面，能避免用户在设置、登录、掌握度等低频页面遇到旧 MVP 质感。
