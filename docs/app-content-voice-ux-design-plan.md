# 面试智练 App 内容状态、语音复述与跨平台体验设计方案

> 日期：2026-05-29  
> 目标：把内容仓库已经具备的结构能力完整落到 App 中，让 Web、手机、桌面都能方便地筛选内容、学习知识、语音复述和复盘练习。

## 1. 背景判断

当前内容仓库已经形成稳定的结构：`manifest -> domains -> categories -> topics`，并支持 `status`、`learningPaths`、`prerequisites`、`recallPrompts.mode`、`interviewFrequency`、`recommendWeight`、`leetcodeUrl`、`rubric.scoreWeights` 等字段。

App 侧已经实现了基础学习闭环：内容加载、目录、详情、文本复述、AI 评分、掌握度和设置。但还有几个关键能力没有完全落地：

1. 设置里不能按身份切换内容阶段，例如发布内容、测试内容、草稿内容。
2. 内容状态只被解析，缺少与 studio 三阶段工作流一致的展示、权限和安全默认策略。
3. 复述练习虽然已有语音输入按钮，但还没有形成完整的跨平台语音复述体验。
4. App 不内置、不提供官方 AI 模型；用户可以配置多个自己的 AI 模型，不同模型能力不同：有的只支持文本，有的支持图片理解，有的支持音频或多模态输入。
5. 复述页需要允许在用户已配置的模型中切换本次使用的模型，并根据模型能力动态开放文本、图片、语音、代码等输入方式。
6. 手机、电脑、网页的输入方式和权限限制不同，需要统一体验但分平台优化。

## 2. 设计原则

1. 内容阶段默认安全：游客只能看正式发布内容；登录用户可在发布/测试之间切换；管理员才可查看草稿。
2. 复述页必须能切换用户自定义 AI 模型：默认使用用户设置的默认模型，但本次练习可以临时选择其他启用模型；没有配置时不显示虚假的内置模型。
3. 语音复述默认转文字：先把语音变成文字，再走文本 AI 评分链路。
4. 图片复述按模型能力开放：模型支持图片理解时可上传图片参与评分；不支持时图片只能作为本地参考或需要先转文字。
5. AI 音频理解是增强能力，不作为基础依赖。
6. 跨平台操作一致：手机适合大按钮、底部操作；桌面适合分栏、快捷键；网页要考虑浏览器权限和兼容。
7. 所有高级能力可降级：麦克风不可用、STT 不可用、AI 不支持音频/图片时，仍然能文本作答。
8. 设置项要真正生效，避免“界面有控件但 provider no-op”。
9. 游客优先：不登录也必须能学习、复述、复习、模拟面试和查看本地统计。
10. 目标可选：不设置目标岗位、面试日期或 JD 也能直接使用；设置后只增强推荐和计划。
11. 渐进采集：不要首次启动就要求用户填写大量信息，目标、JD、项目素材、同步方式都在需要时轻量提示。
12. 能降级就降级：第三方登录、网盘同步、微信绑定、AI 图片/语音等能力未配置时，界面保留入口并提示“功能待开通”，核心本地流程继续可用。
13. 0 成本优先：默认使用本地存储、文件导入导出、用户自带 AI Key 和可选自建同步；需要平台 Key 的能力先完成配置位和降级 UI，不硬编码任何密钥。
14. 隐私默认本地：API Key、JD、项目素材、回答草稿、图片、音频和第三方网盘凭证默认不上传；任何上传、同步或第三方调用都需要明确提示。

## 3. 内容阶段与状态显示设计

### 3.1 Studio 状态工作流

content-studio 已定义三级 topic 状态流转：

```text
draft -> staging -> production
```

状态含义：

| Topic 状态 | 界面文案 | 含义 | 正式用户可见 |
| --- | --- | --- |
| `production` | 已发布 | 正式发布内容 | 是 |
| `staging` | 测试中 | 已提交测试，等待验证或发布 | 否 |
| `draft` | 草稿 | 编辑中或未提交测试 | 否 |
| 缺省 | 未标记 | 兼容旧内容，按草稿或发布取决于 manifest 来源 | 按 manifest 来源判断 |

content-studio 同时维护三个 manifest：

| 内容阶段 | Manifest 文件 | 包含内容 | 使用者 |
| --- | --- |
| 发布内容 | `manifest.json` | `production` | 游客、正式用户 |
| 测试内容 | `staging-manifest.json` | `production + staging` | 登录用户、测试用户 |
| 草稿内容 | `draft-manifest.json` | `production + staging + draft` | 管理员、内容维护者 |

App 侧设置不应该直接让所有人多选任意 topic 状态，而应该按用户身份切换可加载的内容阶段。topic 状态标签用于展示来源和风险。

如果后续新增 `archived`、`reviewing` 等状态，需要同步更新 content schema、App 模型、content-studio 表单、manifest 生成逻辑和校验脚本。

### 3.2 权限与可见范围

| 用户身份 | 可选择内容阶段 | 默认阶段 | 说明 |
| --- | --- | --- | --- |
| 未登录游客 | 发布内容 | 发布内容 | 只能加载 `manifest.json` |
| 普通登录用户 | 发布内容 / 测试内容 | 发布内容 | 可主动切换到测试内容，帮助试用新知识 |
| 管理员 | 发布内容 / 测试内容 / 草稿内容 | 发布内容 | 可查看三阶段内容，用于审核和验证 |

身份判断建议：

1. 游客：本地模式，无 token 或未登录。
2. 普通用户：已登录，但无管理员权限。
3. 管理员：用户信息或 token claims 中包含 `admin` / `content_admin` 角色。

客户端必须做前端隐藏，后端或 Worker 也应做权限保护，避免普通用户通过手改 URL 拉取草稿 manifest。

### 3.3 设置入口

位置：`个人中心 -> 内容与实验功能`

建议新增一个独立设置面板，不放在外观设置里。

设置项：

| 设置项 | 控件 | 默认值 | 说明 |
| --- | --- | --- | --- |
| 内容阶段 | 分段控件 | 发布内容 | 游客只显示发布；普通用户显示发布/测试；管理员显示发布/测试/草稿 |
| 当前身份 | 只读信息 | 游客/用户/管理员 | 帮助用户理解为什么看不到某些阶段 |
| 状态标签 | 开关 | 开 | 测试和草稿内容在目录和详情中显示明显标签 |
| 加载非发布内容确认 | 开关 | 开 | 第一次切换测试/草稿时弹确认说明 |
| 内容版本信息 | 只读信息 | 当前版本 | 显示 contentVersion、领域数、知识点数 |

### 3.4 推荐默认策略

未登录游客：

```text
内容阶段：发布内容
可加载 manifest：manifest.json
可见 topic：production
```

普通登录用户：

```text
内容阶段：发布内容，可切换测试内容
发布内容：manifest.json
测试内容：staging-manifest.json
可见 topic：发布阶段只看 production；测试阶段可看 production + staging
```

管理员：

```text
内容阶段：发布内容，可切换测试内容和草稿内容
发布内容：manifest.json
测试内容：staging-manifest.json
草稿内容：draft-manifest.json
可见 topic：production + staging + draft
```

兼容旧内容的 `missing` 状态不作为用户可选项展示。若某个 topic 从正式 manifest 加载但缺少 `status`，按“已发布”展示；若从草稿 manifest 加载但缺少 `status`，按“草稿”展示，并在内容校验中提示补状态。

### 3.5 页面表现

目录页：

1. 正式内容正常显示。
2. 测试内容在标题旁显示 `测试` 标签。
3. 草稿内容在标题旁显示 `草稿` 标签。
4. 测试和草稿内容按钮使用弱化或边框样式，避免误认为正式内容。
5. 分类计数按当前内容阶段重新计算。

详情页：

1. 标题区域显示状态标签。
2. 测试内容顶部显示提示条：`此知识点为测试内容，可能会继续调整。`
3. 草稿内容顶部显示提示条：`此知识点为草稿内容，可能尚未完成校验。`
4. AI 练习仍可使用，但练习结果要正常记录，避免测试用户和管理员验证不方便。

掌握度页：

1. 默认只统计当前可见状态的内容。
2. 如果用户曾练过测试或草稿内容，但当前切回发布内容，应在底部显示“隐藏了 N 条非发布内容学习记录”。
3. 删除或下线的内容不再推荐，但历史练习记录保留。

### 3.6 数据模型建议

`AppSettings` 增加：

```dart
enum ContentStage { production, staging, draft }

final ContentStage contentStage;
final bool showContentStatusBadge;
final bool confirmBeforeShowingNonProduction;
```

`AuthProvider` 或 `User` 增加：

```dart
enum UserRole { guest, user, admin }

UserRole get effectiveRole;
bool get canUseStagingContent;
bool get canUseDraftContent;
```

`ContentProvider` 增加：

```dart
String manifestPathForStage(ContentStage stage)
List<ContentStage> allowedStagesForRole(UserRole role)
Future<void> switchContentStage(ContentStage stage, UserRole role)
TopicContentStatus effectiveTopicStatus(Topic topic, ContentStage stage)
```

阶段和 manifest 映射：

```text
production -> manifest.json
staging -> staging-manifest.json
draft -> draft-manifest.json
```

如果当前用户失去权限，例如管理员退出登录，App 必须自动切回 `production`。

## 4. AI 模型能力与复述输入设计

### 4.1 多模型配置

App 当前不提供内置模型，也不维护官方模型池。用户可以保存多个自己的 AI 配置，每个配置代表一个可调用模型。配置页除了基础连接信息，还需要声明模型能力，复述页根据用户已有配置决定可用输入方式。

建议 `AiConfig` 增加：

```dart
final bool supportsMultimodal;      // 是否支持多模态输入
final bool supportsImageInput;      // 是否支持图片理解
final bool supportsAudioInput;      // 是否支持直接音频理解
final bool supportsTextInput;       // 默认 true
final bool supportsStreaming;       // 可选，用于后续流式反馈
final Set<String> usageTags;        // recall, mockInterview, imageReview, stt
```

配置页字段：

| 设置项 | 控件 | 默认值 | 说明 |
| --- | --- | --- | --- |
| 模型名称 | 文本输入 | 空 | 例如 `gpt-4o-mini`、`qwen-vl` |
| 启用 | 开关 | 开 | 关闭后不在复述页出现 |
| 默认模型 | 开关 | 关 | 默认评分模型 |
| 支持文本 | 开关 | 开 | 几乎所有模型都应开启 |
| 支持多模态 | 开关 | 关 | 总开关，用于提示模型可接收非文本输入 |
| 支持图片理解 | 开关 | 关 | 开启后复述页可上传图片给该模型 |
| 支持语音输入 | 开关 | 关 | 开启后可直接把音频交给该模型理解 |
| 用途标签 | 多选 | 复述评分 | 复述评分、模拟面试、图片理解、语音识别 |

连接测试建议拆成三类：

1. 文本测试：发送短文本，验证模型可评分。
2. 图片测试：选择一张小图，验证模型可识别图片。
3. 语音测试：录一段短音频，验证模型可转写或理解音频。

如果用户没有测试图片或语音能力，也允许保存，但复述页第一次使用时要给出确认提示。

### 4.2 复述页模型切换

复述页顶部或输入区工具栏需要有模型选择器。模型选择器只展示用户已经配置且启用的模型：

```text
模型：GPT-4o mini [文本]
模型：Qwen VL [文本 图片]
模型：GPT-4o Audio [文本 图片 语音]
```

交互规则：

1. 默认选中 `AiProvider.defaultConfig`。
2. 只显示 `enabled == true` 的配置。
3. 用户本次切换模型只影响当前练习，不一定改变全局默认模型。
4. 可提供“设为默认”入口，但放在菜单里，避免误触。
5. 如果当前答案包含图片，而切换到不支持图片的模型，应提示“该模型不支持图片理解，图片不会发送给 AI”。
6. 如果当前答案包含原始音频，而切换到不支持音频的模型，应自动降级为“先转写再评分”。
7. 如果没有任何启用的 AI 配置，模型选择器显示空状态：“还没有配置 AI 模型”。提供“去配置”入口，同时允许继续进行本地练习。
8. 没有 AI 配置时，复述、复习、模拟面试可降级为本地模式：用户可以看题、作答、保存草稿、查看参考回答和自评，不触发 AI 评分。
9. 需要 AI 的按钮文案应明确，例如“配置 AI 后可获得评分”，而不是让用户点击后才报错。

实现上，`AiProvider.evaluateAnswer` 不应只能使用默认模型，建议增加：

```dart
Future<Map<String, dynamic>> evaluateAnswer({
  required String aiConfigId,
  required String topicId,
  required String question,
  required AnswerPayload answer,
  Rubric? rubric,
})
```

如果没有传 `aiConfigId`，才回退用户设置的默认模型；如果用户没有任何启用的配置，应直接返回“未配置 AI”状态，由页面进入本地练习降级，不发起网络请求。

模型选择器在不同平台的形态：

| 平台 | 推荐控件 |
| --- | --- |
| 手机 | 底部 sheet，显示模型名、能力标签和默认标记 |
| 桌面 | 下拉菜单或侧栏选择器 |
| Web | 下拉菜单，能力不可用时显示 tooltip |

### 4.3 复述输入类型

复述页输入不应只有文本框，建议统一成“答案素材 + 最终答案”的结构。

| 输入类型 | 适用模型 | 基础处理 | 评分输入 |
| --- | --- | --- | --- |
| 文本 | 所有文本模型 | 用户直接输入 | 文本 |
| 语音复述 | 所有模型可用 | 系统 STT 或外部 STT 转文字 | 转写文本 |
| 语音直传 | 支持音频模型 | 原始音频交给模型 | 音频 + 文本指令 |
| 图片上传 | 支持图片模型 | 图片作为答案材料 | 图片 + 文本说明 |
| 图片转文字 | 不支持图片模型 | 用户手动描述或 OCR/STT 服务转写 | 文本 |
| 代码输入 | 文本或多模态模型 | 代码编辑框 | 文本代码 |

建议数据结构：

```json
{
  "selectedAiConfigId": "ai-config-id",
  "answerText": "用户最终答案",
  "attachments": [
    {
      "type": "image",
      "path": "local-or-temp-url",
      "caption": "用户补充说明",
      "sendToAi": true
    },
    {
      "type": "audio",
      "path": "local-temp-audio",
      "transcript": "语音转写文本",
      "sendToAi": false
    }
  ],
  "inputMode": "voice"
}
```

默认不保存原始图片和音频到云端。练习记录默认保存文本、附件摘要和评分结果。

### 4.4 图片复述方案

图片输入适合这些场景：

1. 用户手写/截图了一段代码，让 AI 评价回答是否覆盖关键点。
2. 算法题用户画了思路图或状态转移表。
3. 系统设计题用户上传架构图。
4. 面试官给了一张图，用户根据图复述设计或原理。

交互：

1. 输入区提供图片按钮。
2. 支持粘贴图片、拖拽图片、手机相册/拍照。
3. 图片上传后显示缩略图、删除按钮、可选说明文本。
4. 如果当前模型支持图片理解，缩略图显示“将发送给 AI”。
5. 如果当前模型不支持图片理解，显示“仅本地参考，请用文字描述图片内容”。
6. 提交评估前，如果有图片但模型不支持图片，给出一次明确提示。

跨平台注意：

| 平台 | 图片输入 |
| --- | --- |
| 手机 | 相册、拍照、截图粘贴 |
| 桌面 | 文件选择、拖拽、剪贴板粘贴 |
| Web | 文件选择、拖拽、剪贴板粘贴，注意浏览器文件权限 |

AI 请求策略：

1. 支持图片模型：发送文本题目、rubric、用户文本答案、图片附件。
2. 不支持图片模型：只发送文本答案，图片不上传。
3. Web 端图片请求如果受 CORS 或 body 限制，走 Worker 代理或压缩后发送。

### 4.5 语音复述设计

#### 4.5.1 核心问题

用户想“用嘴复述”，但用户配置的 AI 可能只有文本接口，不支持语音理解。因此不能把语音复述设计为“直接把音频发给 AI 评分”的唯一方案。

推荐采用三层方案：

1. **系统语音转文字（默认）**：使用设备或浏览器的 STT，把语音变成文字，再调用现有文本 AI 评分。
2. **外部 STT 服务（可选）**：用户配置支持音频转写的服务，例如 Whisper 兼容接口。
3. **AI 音频理解（高级可选）**：如果用户配置的模型支持音频输入，可以直接发送音频并要求转写和评估。

默认体验必须基于第 1 层。

#### 4.5.2 推荐流程

```text
用户点击麦克风
-> App 请求麦克风权限
-> 用户开始复述
-> 实时/结束后转文字
-> 用户编辑转写文本
-> 点击 AI 评估
-> AI 按 rubric 对文本答案评分
-> 更新掌握度和复习计划
```

关键点：转写文本必须允许用户编辑。语音识别会出错，尤其是技术术语、英文缩写、代码名词。

#### 4.5.3 复述页交互

复述区域建议分为四种输入模式：

| 模式 | 适合场景 | 控件 |
| --- | --- | --- |
| 文本 | 普通键盘输入 | 多行文本框 |
| 语音 | 口头复述 | 大号麦克风按钮 + 转写文本框 |
| 图片 | 截图、手写图、架构图、算法图 | 图片缩略图 + 补充说明 |
| 代码 | `recallPrompts.mode=code` | 等宽代码输入框 |

移动端布局：

1. 题目在上方，输入区域占主要空间。
2. 底部固定操作条：麦克风、清空、提交评估。
3. 录音时显示大号状态：正在听、时长、声波或脉冲动画。
4. 结束录音后自动滚动到转写文本。

桌面布局：

1. 左侧题目、rubric 提示、相关知识点。
2. 右侧输入区和 AI 评估结果。
3. 麦克风按钮放在输入框工具栏右侧。
4. 支持快捷键：`Cmd/Ctrl + Enter` 提交，`Cmd/Ctrl + M` 开始/停止录音。

Web 布局：

1. 首次点击麦克风时触发浏览器权限。
2. 权限失败时显示浏览器设置引导。
3. Safari/Firefox 不支持某些语音能力时，降级为文本输入。

#### 4.5.4 语音能力分层

##### A. 系统 STT 默认方案

当前项目已使用 `speech_to_text`。这条路线的优点是：

1. 不要求用户 AI 支持音频。
2. 不需要把音频上传给第三方 AI，隐私压力更小。
3. 跨 Android、iOS、macOS、Web 有现成插件路线。

局限：

1. Web 端依赖浏览器实现，兼容性不一致。
2. 技术术语识别可能不稳定。
3. 某些桌面平台需要额外权限配置。

适合做默认能力。

##### B. 外部 STT 服务可选方案

新增一个“语音识别配置”：

| 设置项 | 说明 |
| --- | --- |
| 识别方式 | 系统识别 / 自定义 STT / AI 音频理解 |
| STT Base URL | OpenAI Whisper 兼容接口或其他服务 |
| STT API Key | 仅本地保存 |
| STT Model | 例如 `whisper-1` |
| 语言 | 自动 / 中文 / 英文 |

流程：

```text
录音 -> 保存临时音频 -> 调 STT 服务 -> 得到文本 -> 用户编辑 -> 文本 AI 评分
```

注意：这仍然不要求评分 AI 支持音频。STT 配置和 AI 评分配置可以是两套。

##### C. AI 音频理解高级方案

只有当用户明确配置支持音频的模型时启用，例如多模态实时或音频模型。

流程：

```text
录音 -> 发送音频 + rubric -> AI 返回转写文本和评分 JSON
```

界面要显示“该模式会上传原始音频”提示。

不建议第一阶段默认启用，原因：

1. 不同模型 API 格式差异大。
2. 成本更高。
3. 隐私提示更复杂。
4. 用户现有 OpenAI 兼容文本配置未必可用。

#### 4.5.5 语音复述结果结构

建议练习记录保存：

```json
{
  "topicId": "java.jvm.runtime-data-area",
  "promptId": "java.jvm.runtime-data-area.recall.1",
  "inputMode": "voice",
  "transcript": "用户语音转写后的文本",
  "editedTranscript": "用户编辑后的最终文本",
  "score": 82,
  "sttProvider": "system",
  "audioSaved": false,
  "startedAt": "...",
  "completedAt": "..."
}
```

默认不保存原始音频，只保存文本和评分摘要。若后续支持保存音频，必须单独设置并明确提示。

#### 4.5.6 语音复述失败降级

| 失败场景 | 降级策略 |
| --- | --- |
| 没有麦克风权限 | 显示权限说明，保留文本输入 |
| 当前平台 STT 不可用 | 提示切换文本输入或配置自定义 STT |
| 转写为空 | 允许重录，显示录音时长和权限状态 |
| 技术术语识别错误 | 允许编辑转写文本，提供常用术语快速替换 |
| AI 评分失败 | 保留草稿答案，可重试 |
| 网络不可用 | 保存本地练习草稿，稍后评估 |

## 5. AI 配置、模型能力与语音配置关系

建议拆成两类配置：AI 模型配置和语音识别配置。其中 AI 模型配置可以有多个，每个模型都要声明自己的输入能力。

### 5.1 AI 模型配置

用于评分、纠错、图片理解、音频理解等：

| 字段 | 说明 |
| --- | --- |
| Base URL | OpenAI 兼容接口 |
| API Key | 本地保存 |
| Model | 模型名 |
| Enabled | 是否启用 |
| Default | 是否默认模型 |
| Supports Text | 是否支持文本输入，默认 true |
| Supports Image | 是否支持图片理解 |
| Supports Audio | 是否支持直接音频输入 |
| Supports Multimodal | 是否支持多模态组合输入 |
| Usage Tags | 适用用途：复述评分、模拟面试、图片理解、语音识别 |

能力声明不是装饰字段，会直接影响复述页 UI：

1. 不支持图片的模型：隐藏或禁用“发送图片给 AI”。
2. 不支持音频的模型：语音只走 STT 转文本，不展示“音频直传”。
3. 只支持文本的模型：仍可使用文本、语音转写文本、代码文本。
4. 支持图片的模型：可上传截图、架构图、手写图参与评分。
5. 支持音频的模型：可选择“转文字后评分”或“音频直传评分”。

### 5.2 语音识别配置

用于音频转文字。它和 AI 评分模型可以相互独立：

| 字段 | 说明 |
| --- | --- |
| Provider | 系统 / 自定义 STT / AI 音频 |
| Base URL | 自定义 STT 时填写 |
| API Key | 自定义 STT 时填写 |
| Model | STT 模型 |
| Language | 默认中文，可自动识别 |

### 5.3 用户文案

在设置页中明确说明：

```text
语音复述默认会先转写成文字，再使用你选择的 AI 模型进行评分。
因此，即使当前模型不支持音频，也可以使用语音复述。
如果模型支持图片理解，你也可以在复述中上传截图、手写图或架构图。
```

如果启用 AI 音频模式：

```text
该模式会把原始音频发送给 AI 服务，可能产生更高费用。请确认你的模型支持音频输入。
```

## 6. 跨平台体验设计

### 6.1 手机端

重点：单手操作、按钮大、状态清楚。

复述页：

1. 题目卡片简洁，只保留标题、题目、预计分钟、难度。
2. 顶部显示当前模型，点击后用底部 sheet 切换模型。
3. 输入区默认显示文本、语音、图片、代码入口；不支持的能力置灰并说明原因。
4. 麦克风按钮固定在底部中间，直径建议 56-64。
5. 图片入口放在底部工具条，支持拍照、相册和截图粘贴。
6. 录音状态使用醒目的进度环或脉冲，不使用太多文字说明。
7. 评估结果用分段卡片：分数、遗漏点、错误点、优化回答、下一步。

目录页：

1. 领域切换使用横向 chips 或下拉，不用挤满的 segmented button。
2. topic 操作使用图标按钮或底部 sheet，避免两个长文字按钮撑爆列表。
3. 搜索放在顶部，筛选放在 bottom sheet。

### 6.2 桌面端

重点：信息密度、分栏、快捷键。

学习详情：

1. 左侧目录或章节导航。
2. 中间知识内容。
3. 右侧复述面板或掌握度摘要。

复述页：

1. 左题目右输入。
2. 输入区顶部固定模型选择器和能力标签。
3. 图片支持拖拽、文件选择和剪贴板粘贴。
4. AI 结果可固定在右侧或下方。
5. 支持快捷键。
6. 多题练习显示题目列表和答题状态。

### 6.3 Web 端

重点：浏览器权限、兼容和 CORS。

语音：

1. 首次录音前显示轻量权限提示。
2. 权限失败后提供文本输入，不阻断练习。
3. 自定义 STT 如果存在 CORS 问题，走 Worker 代理或提示用户使用 App 端。

图片：

1. 支持文件选择、拖拽和剪贴板粘贴。
2. 上传前压缩大图，避免请求体过大。
3. 当前模型不支持图片时，不上传图片，只作为本地参考。

AI：

1. Web 端 AI 调用继续优先通过 Worker 代理。
2. 用户自定义 Base URL 的 CORS 风险要在配置测试时提示。

## 8. 信息架构调整

### 8.1 个人中心

建议分组：

1. 账户管理：资料展示、头像、昵称、账号 ID、登录状态、本地游客身份、绑定邮箱、绑定微信、绑定其他账号。
2. 数据统计：练习天数、完成题数、平均分、最近趋势、同步状态。
3. 同步状态：当前同步方式、上次同步时间、待同步数量、最近失败原因。
4. 快捷入口：导出数据、同步设置、AI 配置、外观设置、关于与更新。
5. AI 模型配置：多个模型、默认模型、能力声明、连接测试。
6. 语音识别配置：系统 STT、自定义 STT、AI 音频模型。
7. 内容与实验功能。
8. 外观与主题。
9. 学习策略：预设策略、自定义权重、每日新学/复习数量、模拟面试组卷偏好。
10. 关于与更新。

个人中心只放账号、统计、绑定和数据状态；具体配置项放到设置页的对应分组中。未登录用户也能进入个人中心，展示“本地游客模式”、本地统计和导出入口，并提示登录或同步只是为了跨设备备份。

### 8.1.1 账户管理模块

账户管理是个人中心的首屏模块，负责展示身份和所有账号绑定关系。

展示区：

1. 头像：未登录时使用本地头像或默认头像；登录后可同步账号头像。
2. 昵称：未登录时可编辑本地昵称；登录后可修改账号昵称。
3. 身份状态：本地游客 / 已登录 / 管理员。
4. 账号 ID：已登录时显示；未登录时显示本地设备 ID 或本地资料 ID。
5. 数据状态：本地数据量、上次备份时间、待同步数量。

绑定区：

| 绑定项 | 状态 | 操作 | 降级提示 |
| --- | --- | --- | --- |
| 邮箱 | 未绑定 / 已绑定 / 待验证 / 待开通 | 绑定、换绑、解绑、重新发送验证 | 邮件服务未配置时显示“邮箱绑定待开通” |
| 微信 | 未绑定 / 已绑定 / 待开通 | 绑定、解绑 | 微信 AppID 未配置时显示“微信绑定待开通” |
| 其他账号 | 未绑定 / 已绑定 / 待开通 | 预留 GitHub、Apple、Google 等入口 | 对应 OAuth Key 未配置时显示“暂未开通” |

行为规则：

1. 账号绑定不是使用 App 的前置条件。
2. 未登录用户可以编辑本地头像和昵称，登录后提示是否合并到账号资料。
3. 绑定邮箱、绑定微信、绑定其他账号都从个人中心的账户管理进入，不散落在设置页。
4. 设置页只保留功能配置，例如同步方式、AI 模型、外观、学习策略。
5. 第三方绑定未配置平台 Key 时，按钮保持可见但不可执行，并解释当前可用替代方式，例如“可先使用本地数据和 WebDAV 备份”。

### 8.1.2 无目标、无登录的默认体验

默认体验必须是开箱即用：

1. 首次启动不要求登录、不要求填写目标岗位、不要求绑定同步。
2. 首页默认基于当前领域、学习进度、复习到期、高频权重给出下一步。
3. 面试准备页在未设置目标时显示“通用技术面试准备”模式。
4. 用户设置目标岗位、面试日期、JD 后，才启用倒计时计划、岗位匹配和就绪度增强解释。
5. 未登录时所有训练数据写入本地；登录后再提示合并本地数据到账号。

### 8.1.3 同步与备份方式

同步设计按 0 成本和可用性分层：

| 方式 | 是否默认可用 | 成本 | 适合场景 | 降级策略 |
| --- | --- | --- | --- | --- |
| 本地存储 | 是 | 0 | 单设备日常使用 | 无需降级 |
| 文件导出/导入 | 是 | 0 | 手动备份、换设备迁移 | 无云服务时作为兜底 |
| 自定义 WebDAV | 是，用户自填地址和账号 | 0 平台成本 | 坚果云、NAS、Nextcloud、群晖等 | 连接失败则保留本地待同步 |
| App 账号云同步 | 已登录后可用 | Cloudflare 免费额度内 | 多端自动同步 | Worker/D1 不可用时转本地队列 |
| 百度网盘 | 需要平台 Key 和用户授权 | 可能需要审核 | 国内用户备份 | 未配置 Key 时显示待开通 |
| 夸克网盘 | 需要平台 Key 或开放接口能力 | 取决于平台 | 国内用户备份 | 未配置 Key 时显示待开通 |
| 阿里云盘 | 需要平台 Key 或开放接口能力 | 取决于平台 | 国内用户备份 | 未配置 Key 时显示待开通 |
| OneDrive / Google Drive | 需要 OAuth Key | 可能需要审核 | 海外或跨平台用户 | 未配置 Key 时显示待开通 |

首轮优先做本地存储、文件导入导出、自定义 WebDAV 和现有账号云同步。百度网盘、夸克网盘、阿里云盘、OneDrive、Google Drive 先完成设置入口、授权状态、待开通提示和配置占位，不硬编码 Key。

### 8.1.4 第三方 Key 与隐私配置

所有第三方能力都不能在代码中写死密钥，也不能提交到仓库。需要预留配置位：

1. Worker 侧密钥：通过 Cloudflare Worker secrets 或部署环境变量配置。
2. Flutter 构建配置：只放公开 client id 或开关；敏感 secret 必须走后端代理。
3. 本地开发：使用 `.env` 或本机配置文件，并加入 `.gitignore`。
4. 用户自填凭证：WebDAV 地址、用户名、应用密码只保存在本地；如需同步必须端到端加密。
5. 未配置平台 Key 的功能：按钮可见但不可执行，提示“该同步/绑定方式暂未开通，可先使用本地备份或 WebDAV”。

后续统一配置时需要准备的 Key：

| 能力 | 配置项 | 建议位置 | 未配置时表现 |
| --- | --- | --- | --- |
| 微信绑定/登录 | `WECHAT_APP_ID`、`WECHAT_APP_SECRET`、`WECHAT_REDIRECT_URI` | Worker secrets / 部署环境变量 | 显示微信绑定待开通 |
| 邮箱绑定/验证码 | `MAIL_PROVIDER`、`MAIL_API_KEY`、`MAIL_FROM` | Worker secrets | 显示邮箱服务待开通，允许本地资料继续编辑 |
| 百度网盘 | `BAIDU_NETDISK_CLIENT_ID`、`BAIDU_NETDISK_CLIENT_SECRET`、`BAIDU_NETDISK_REDIRECT_URI` | Worker secrets | 显示百度网盘同步待开通 |
| 夸克网盘 | `QUARK_NETDISK_CLIENT_ID`、`QUARK_NETDISK_CLIENT_SECRET`、`QUARK_NETDISK_REDIRECT_URI` | Worker secrets | 显示夸克网盘同步待开通 |
| 阿里云盘 | `ALIYUN_DRIVE_CLIENT_ID`、`ALIYUN_DRIVE_CLIENT_SECRET`、`ALIYUN_DRIVE_REDIRECT_URI` | Worker secrets | 显示阿里云盘同步待开通 |
| OneDrive | `MICROSOFT_CLIENT_ID`、`MICROSOFT_CLIENT_SECRET`、`MICROSOFT_REDIRECT_URI` | Worker secrets | 显示 OneDrive 同步待开通 |
| Google Drive | `GOOGLE_CLIENT_ID`、`GOOGLE_CLIENT_SECRET`、`GOOGLE_REDIRECT_URI` | Worker secrets | 显示 Google Drive 同步待开通 |
| WebDAV | 用户自填 WebDAV URL、用户名、应用密码 | 本地加密存储 | 不需要平台 Key，连接失败则保留本地数据 |
| AI 代理 | `AI_PROXY_ALLOWED_HOSTS`、可选服务端代理密钥 | Worker secrets | 回退用户本地 AI 配置或提示配置 AI |

### 8.2 练习页

建议分为：

1. 今日复习。
2. 路线练习。
3. 随机抽问。
4. 代码复述。
5. 模拟面试。
6. 追问训练。

今日复习需要从入口升级为独立工作台：

1. 到期复习、逾期复习、低分重练、错题回流、高频未掌握分组展示。
2. 每个复习项显示推荐原因、上次得分、上次遗漏点、预计耗时和建议输入模式。
3. 支持开始复习、跳过到明天、降低难度、加入模拟面试题池。
4. 完成后更新掌握度、下次复习时间，并生成当天复习小结。

模拟面试需要从连续题升级为面试房间：

1. 场景选择：基础知识、项目深挖、系统设计、代码题、行为面试、混合面试。
2. 题目流程：主问题、用户回答、AI 追问、澄清、总结。
3. 面试中展示计时器、面试官关注点、草稿区、回答输入区和当前进度。
4. 面试结束后生成报告：总分、维度评分、薄弱知识、表达问题、关键回放、下一轮训练包。

### 8.2.1 学习策略

学习策略决定 App 如何推荐“下一步学什么、今天复习什么、模拟面试抽哪些题”。它不是必填项，默认策略必须足够好用；高级用户可以在设置页自定义。

默认预设：

| 策略 | 适合用户 | 推荐逻辑 |
| --- | --- | --- |
| 智能推荐 | 大多数用户 | 综合低分、到期复习、高频、路径顺序和最近练习情况 |
| 低分优先 | 已经刷过一轮的人 | 优先补不熟练、未掌握、错因明显的 topic |
| 路径顺序 | 新手或系统学习 | 按 `learningPaths`、分类顺序和 `order` 推进 |
| 高频优先 | 临近面试冲刺 | 按 `interviewFrequency`、`recommendWeight` 和掌握度排序 |
| 复习优先 | 长期使用者 | 优先处理到期、逾期、低分回流和错题回流 |

可自定义项：

1. 每日新学数量。
2. 每日复习数量。
3. 推荐权重：低分、逾期、高频、路径顺序、最近未练、前置知识。
4. 是否优先高频题。
5. 是否优先补前置知识。
6. 是否允许跳过低频 topic。
7. 模拟面试组卷偏好：基础知识、系统设计、代码题、项目深挖、混合。
8. 临近面试时是否自动提高高频题和模拟面试权重。

降级规则：

1. 用户不配置学习策略时，使用智能推荐。
2. 用户未设置目标岗位或面试日期时，不启用岗位/JD 权重，只使用通用策略。
3. 没有 AI 配置时，仍然可以按本地分数、练习次数、复习时间和内容权重推荐。
4. 自定义策略如果配置异常，回退智能推荐，并提示用户检查设置。

### 8.3 目录页

建议加入：

1. 搜索。
2. 内容阶段提示和状态标签。
3. 高频筛选。
4. 难度筛选。
5. 学习路径 / 分类列表切换。

### 8.4 全量页面视觉一致性

原有页面都要跟随新的工作台视觉方案一起打磨：

1. 学习首页：今日任务、路线、薄弱点、下一步最佳行动。
2. 知识目录：搜索、筛选、状态标签、高频和掌握度信息。
3. 知识详情：桌面三栏，移动端分步；面试官关注点、前置知识和回答模板突出展示。
4. 复述页：文本、语音、图片、代码统一输入系统。
5. 今日复习：独立复习队列和复习结果小结。
6. 模拟面试：面试场景、追问流、计时和报告。
7. 掌握度：从静态看板升级为诊断和行动入口。
8. 个人中心、登录、AI 配置、更新：统一表单、标签、空状态、错误状态和说明文案。

## 9. 实现拆分计划

### P0：内容阶段设置

1. `AppSettings` 增加 `contentStage` 配置。
2. `AuthProvider/User` 增加角色判断：游客、普通用户、管理员。
2. 个人中心新增“内容与实验功能”面板。
3. 游客只显示发布内容；普通用户可切换发布/测试；管理员可切换发布/测试/草稿。
4. `ContentProvider` 按内容阶段加载对应 manifest。
5. 目录、首页、掌握度、练习入口统一使用当前阶段 topic。
6. 测试和草稿 topic 增加状态标签和提示条。

验收：

1. 未登录时只能看到发布内容，无法切换测试或草稿。
2. 普通登录用户可以在发布和测试之间切换。
3. 管理员可以看到发布、测试、草稿三个阶段。
4. 切换阶段后目录、首页、掌握度、练习题池同步变化。
5. 测试和草稿内容有清晰标签和风险提示。

### P1：复述页模型切换与基础语音体验

1. `AiConfig` 增加模型能力字段：文本、图片、音频、多模态、用途标签。
2. AI 配置页支持编辑能力字段，并在配置卡片显示能力标签。
3. 复述页增加模型选择器，只展示用户自定义且启用的模型；默认使用用户设置的默认模型，本次练习可临时切换。
4. 复述页增加输入模式切换：文本 / 语音 / 图片 / 代码。
5. 基于现有 `speech_to_text` 做系统 STT 默认方案。
6. 录音结果进入可编辑文本框。
7. AI 评分默认仍走文本评估。
8. 根据所选模型能力动态启用或禁用图片、音频直传入口。
9. 没有 AI 配置时进入本地练习模式：可看题、作答、保存草稿、查看参考回答和自评，并提示配置 AI 后可评分。
10. AI 调用失败时保留文本输入、附件和所选模型，允许切换配置后重试。

验收：

1. 复述页可以切换用户启用中的 AI 模型。
2. 模型不支持音频时，也能用语音转文字完成复述。
3. 语音转写文本可编辑。
4. 模型不支持图片时，图片入口清楚提示不可发送给 AI。
5. 没有 AI 配置时，页面不显示虚假的模型选项，且本地练习流程可继续。
6. 手机和桌面布局都不拥挤。

### P2：自定义 STT 配置

1. 新增语音识别配置模型。
2. 支持系统 STT / 自定义 STT 切换。
3. 支持 STT 连接测试。
4. Web 端处理 CORS 提示或 Worker 代理。

验收：

1. 用户可使用 Whisper 兼容服务转写。
2. STT 配置和评分 AI 配置相互独立。
3. 配置失败时能回退系统 STT 或文本输入。

### P3：图片复述与多模态评分

1. 复述页支持图片附件：拍照、相册、文件选择、拖拽、剪贴板粘贴。
2. 图片上传后显示缩略图、删除按钮、补充说明和“是否发送给 AI”状态。
3. 当前模型支持图片理解时，把图片和文本答案一起发送给 AI。
4. 当前模型不支持图片理解时，图片只作为本地参考，不上传给 AI。
5. Web 端图片自动压缩，必要时走 Worker 代理。

验收：

1. 支持图片模型可以基于图片 + 文本进行评分。
2. 不支持图片模型不会误传图片。
3. 手机和桌面都能方便添加图片。

### P4：今日复习工作台

1. 用当前 `nextReviewAt`、`reviewCount`、最近得分和 topic 高频信息生成复习队列。
2. 复习队列按原因分组：到期、逾期、低分、错题、高频未掌握。
3. 复习卡展示推荐原因、上次得分、上次遗漏点、预计耗时、建议模式。
4. 支持开始复习、跳过到明天、降低难度、加入模拟面试题池。
5. 完成复习后展示今日小结，并更新掌握度和下次复习时间。

验收：

1. 用户能清楚知道今天为什么要复习这些知识点。
2. 复习完成后能看到通过、待重练、已推迟和加入面试题池的结果。
3. 首页、掌握度、练习页看到的待复习数量一致。
4. 没有到期内容时有下一步建议，而不是空白页。

### P5：模拟面试房间与报告

1. 增加面试场景选择：基础知识、项目深挖、系统设计、代码题、行为面试、混合面试。
2. 支持从今日复习、薄弱点、高频题、学习路径生成一场模拟面试。
3. 面试房间展示题目、计时、回答输入、追问、草稿、面试官关注点和进度。
4. AI 根据用户回答生成追问，追问数量可限制，避免无限对话。
5. 结束后生成面试报告：总分、维度评分、知识薄弱点、表达问题、关键问题回放、下一轮训练包。

验收：

1. 用户可以在 10-30 分钟内完成一场结构化模拟面试。
2. 面试题不是固定裸列表，而有主问、追问和总结。
3. 报告能直接生成下一轮复习或追问训练入口。
4. 移动端能单题沉浸答题，桌面端能左右分栏复盘。

### P6：内容路线与练习增强

1. 解析 `learningPaths`。
2. 路径视图使用真实 steps，而不是 `phase` 兜底。
3. 展示 topic `prerequisites`，支持前置补学。
4. 复述练习支持多个 prompt 和 `mode=code`。
5. LeetCode 链接在算法详情页展示。

验收：

1. Java、Agent、算法等领域能看到内容端维护的真实路线。
2. 代码题进入代码输入模式。
3. 算法题可一键打开 LeetCode。

### P7：面试准备功能打磨

1. 高频题冲刺：按 `interviewFrequency`、`recommendWeight`、掌握度和复习逾期程度排序。
2. 追问训练：优先使用内容中的 `followUps`，缺失时由 AI 基于 topic 动态生成。
3. 项目深挖：提供背景、技术决策、取舍、指标、故障复盘、个人贡献输入框；模板、草稿和记录都归用户侧本地数据。
4. 系统设计：提供需求澄清、容量估算、架构拆分、瓶颈和权衡输入框；用户答案和训练过程归用户侧本地数据。
5. 代码题：支持思路、复杂度、边界条件、测试用例和 LeetCode 链接。
6. 准备日程：目标岗位、面试日期、每日新学/复习/模拟面试建议量。
7. JD/岗位描述解析：用户粘贴岗位描述后，结合现有 topic 生成准备重点和缺口。
8. 回答版本库：保存高频题的用户初稿、AI 修改稿和最终面试版。
9. 提示阶梯：答不上来时按方向提示、关键词提示、结构提示、参考回答逐级展开。
10. 未设置目标岗位或 JD 时，使用通用面试准备模式，不阻断任何练习入口。

验收：

1. 用户可以围绕一次真实面试建立准备计划。
2. 高频题、项目、系统设计、代码题都有清晰入口和反馈方式。
3. 所有训练结果都能回流到掌握度和复习队列。
4. 用户能看到自己的面试就绪度、倒计时计划和下一步最该练什么。
5. 不填写目标岗位、面试日期或 JD 的用户也能完整使用学习、复习和模拟面试。

### P8：面试就绪度与复盘体系

1. 增加面试准备首页：目标岗位、面试日期、倒计时、今日计划、就绪度。
2. 未设置目标时显示通用就绪度；设置目标后再加入岗位匹配、倒计时和 JD 缺口。
3. 就绪度由掌握度、复习逾期、模拟面试均分、低分 topic、表达维度趋势综合计算。
4. 练习记录保存更完整的维度结果：问题、用户回答、分数、遗漏点、错误点、优化回答、下一步。
5. 错因标签归类：概念缺失、混淆、表达不清、缺少场景、深度不足、代码边界遗漏。
6. 模拟面试报告支持按题回放，并能一键生成复习队列或下一场训练包。
7. 正式模拟模式默认结束后统一反馈；练习模式可以逐题即时反馈。

验收：

1. 用户能判断自己离目标面试还有多远，而不是只看到零散分数。
2. 任意一次低分练习都能追溯原因，并进入下一轮训练。
3. 面试报告能转化成可执行的今日复习、追问训练或模拟面试。
4. 没有目标设置时，就绪度仍基于通用面试准备维度可用。

### P9：账号、同步与配置中心

1. 个人中心新增账户管理模块，展示本地游客模式、账号信息、头像、昵称、账号 ID、绑定状态、统计概览和同步状态。
2. 支持修改用户信息：头像、昵称、邮箱展示名等；未登录时保存在本地资料中。
3. 账号绑定入口全部放在账户管理：邮箱、微信、其他账号；后续可扩展手机号、GitHub、Apple、Google。
4. 设置页增加同步与备份分组：本地导出/导入、自定义 WebDAV、账号云同步、百度网盘、夸克网盘、阿里云盘、OneDrive、Google Drive。
5. 本地导出/导入和 WebDAV 优先落地；网盘 OAuth 类能力先完成 UI、状态机、配置占位和降级提示。
6. 所有第三方密钥走环境变量、Worker secrets 或本机未入库配置，不在代码和文档中写真实 Key。
7. 未登录用户也能使用本地统计、导出、WebDAV；账号云同步和账号绑定作为增强能力。
8. 平台 Key 未配置时，账户管理里的对应绑定项显示“待开通”，并给出本地使用和 WebDAV 备份替代路径。

验收：

1. 不登录可以完整使用核心学习和本地统计。
2. 用户可以通过文件导出/导入迁移数据。
3. 用户可以配置 WebDAV 完成 0 平台成本同步。
4. 第三方网盘、微信绑定、邮箱服务未配置时不会报错或阻断流程，只显示待开通提示。
5. 代码仓库不包含任何真实 AppID、Secret、Token 或测试账号凭证。
6. 绑定邮箱、绑定微信、绑定其他账号只能从个人中心账户管理进入；设置页不重复放绑定入口。

### P10：视觉优化与跨平台细节

1. 移动端底部录音操作条。
2. 桌面端复述分栏。
3. 目录页搜索和筛选。
4. 设置页 no-op 项全部落地，包括字体大小、卡片密度、同步。
5. 增加平台能力检测和友好提示。
6. 登录、个人中心、AI 配置、掌握度、更新页统一视觉组件。
7. 所有原有页面补齐加载、空状态、错误状态、离线状态。

验收：

1. 手机端单手可完成复述。
2. 桌面端可高效刷题和复盘。
3. Web 端权限失败不阻塞核心练习。
4. 用户在任意原有页面都不会看到旧 MVP 质感或无效设置项。

## 10. 知识契约是否需要改变

结论：本轮不需要改知识契约。

依据 content 项目现有文档和 schema，初版改进需要的公共知识字段已经存在：

| 需求 | 使用现有字段 |
| --- | --- |
| 复述题 | `recallPrompts.id`、`recallPrompts.prompt`、`recallPrompts.mode`、`recallPrompts.expectedMinutes`、`recallPrompts.difficulty` |
| 面试追问 | `learningCards.type=interviewAnswer` 里的 `followUpQuestions` |
| 面试回答模板 | `learningCards.type=interviewAnswer` |
| 高频和推荐 | `interviewFrequency`、`recommendWeight` |
| 前置补学 | `prerequisites` |
| 面试官关注点 | `interviewerFocus` |
| 学习耗时 | `estimatedMinutes` |
| 评分和错因归类 | `rubric.mustHave`、`rubric.goodToHave`、`rubric.commonMistakes`、`rubric.scoreWeights` |

只有出现字段更名、删除字段、增加必填字段、修改字段类型/语义/枚举、增加新的 `learningCards.type` 或 `recallPrompts.mode` 时，才算知识契约变更。本轮计划不需要这些改动。

## 11. 风险与注意事项

1. Web 语音识别兼容性不稳定，必须保留文本输入。
2. 原始音频涉及隐私，默认不保存、不上传。
3. 图片可能包含隐私信息，默认只在模型支持图片且用户确认时上传。
4. 自定义 STT 和多模态模型可能产生额外费用，设置页要明确提示。
5. 模型能力字段依赖用户声明，连接测试应尽量验证真实能力。
6. 内容阶段切换会影响掌握度统计，所有页面必须使用同一阶段数据。
7. 测试或草稿内容如果参与练习，后续内容删除时要保留历史进度但不再推荐。
8. 草稿 manifest 必须做权限保护，不能只靠前端隐藏。
9. 不建议第一阶段直接做 AI 音频理解，先把“语音转文字 + 文本评分”打磨好。
10. 模拟面试如果完全依赖 AI 动态追问，需要限制轮数、超时和费用。
11. 复习队列和模拟面试都写入掌握度时，要避免一次面试结果过度覆盖长期学习记录。
12. 项目深挖和行为面试可能涉及用户隐私，默认只保存摘要，不保存完整敏感回答。
13. 若未来真的发生知识契约变更，要保证旧内容仍可渲染和练习。
14. 目标岗位、JD、面试日期都是增强信息，不能成为使用 App 的前置条件。
15. 第三方同步和账号绑定依赖平台 Key、审核或开放接口，必须提供未开通降级提示。
16. 百度网盘、夸克网盘、阿里云盘等平台能力可能变化，首轮不能把它们作为唯一同步方式。
17. 第三方 AppID、Secret、OAuth 回调密钥、邮件服务 Key、网盘 API Key 不能写入仓库；部署时统一配置。

## 12. 最小可落地版本

第一轮建议只做以下内容：

1. 设置中新增内容阶段切换。
2. 按身份控制内容阶段：游客发布、普通用户发布/测试、管理员发布/测试/草稿。
3. 全局按当前阶段加载对应 manifest。
4. 目录和详情显示测试/草稿标签。
5. AI 配置支持用户自定义模型能力声明：文本、图片、语音、多模态。
6. 复述页可以在用户已配置的模型中切换本次使用的 AI 模型；没有配置时显示配置提示和本地练习降级。
7. 复述页加入语音模式，使用系统 STT 转文字。
8. 转写文本可编辑，继续走文本 AI 评分。
9. 图片入口先做可见能力提示；支持图片模型时允许图片参与评分。
10. 移动端优化录音按钮，桌面端保留输入框工具栏按钮。
11. 今日复习页展示真实复习队列、复习原因、上次得分和完成小结。
12. 模拟面试页支持场景选择、主问题、有限追问、计时和结束报告。
13. 首页、目录、详情、复述、复习、模拟面试、掌握度、个人中心全部套用统一视觉组件。
14. 无需改知识契约，先用现有字段实现；用户侧训练数据单独走本地库和同步模型。
15. 未登录、未设置目标岗位、未配置第三方同步时，核心学习和练习仍完整可用。
16. 设置页提供本地导出/导入、WebDAV 配置、账号云同步和第三方网盘占位。
17. 微信绑定、邮箱绑定、百度网盘、夸克网盘等未配置平台 Key 时，显示“功能待开通”而不是隐藏或报错。

这一轮完成后，用户就能：

1. 在 App 里按身份决定加载发布内容、测试内容还是草稿内容。
2. 登录后能切换测试内容，管理员能审核草稿内容。
3. 在复述时临时选择本次使用的用户自定义 AI 模型；未配置 AI 时也能继续本地练习。
4. 在不要求 AI 支持音频的情况下，用语音完成复述。
5. 在模型支持图片时，用截图、手写图、架构图参与复述评分。
6. 在手机、桌面、网页上都保留可用的练习路径。
7. 每天打开 App 就知道该复习什么、为什么复习、复习完下一步做什么。
8. 能围绕薄弱点或高频题完成一场初版模拟面试，并拿到可执行的复盘报告。
9. 不登录也能持续使用，并通过文件或 WebDAV 备份本地数据。
10. 需要账号绑定或第三方网盘时，能看到清楚的开通状态和配置提示。
